# Specs (`docs/specs/`)

Place **feature specifications** here as `*.spec.md`: acceptance criteria, links to tickets, and pointers to `docs/context/` for architecture.

For larger initiatives you may add a sibling `*.implementation-plan.md` in the same folder (e.g. `docs/specs/<topic>/`) to track phased repo work and Cursor IDE checklists without mixing normative requirements into the spec.

Referenced by the Eversis **Implement** workflow and `.cursor/prompts/public/eversis-implement.md`.

# AI Devs | s02e05 — zadanie `drone`

## Cel

Zbudować sekwencję poleceń sterujących dronem **DRN-BMB7** i zweryfikować ją na hubie kursowym.

## Deliverable

Wysyłka do `/verify`:

```json
{
  "apikey": "<HUB_API_KEY>",
  "task": "drone",
  "answer": {
    "instructions": ["...", "..."]
  }
}
```

Po poprawnej sekwencji hub zwraca flagę `{FLG:...}`.

## Źródła danych

| Zasób | URL / lokalizacja |
| --- | --- |
| Dokumentacja API (live) | `https://hub.ag3nts.org/dane/drone.html` — agent pobiera przez `http_request` |
| Snapshot docs (autorzy) | [`drone-api.md`](./drone-api.md) — **nie** dla agenta w runtime |
| Mapa PNG | Prefetch bootstrap → `data/map.png` (z `DRONE_MAP_URL` w `tasks/.env`) |

## Kluczowe kroki (homework)

1. **Vision** — `analyze_image_vision` na **lokalnym** pliku mapy (prefetch w `run.ts`, nie przez agenta).
2. **Live docs** — pobranie HTML i uważne czytanie (pułapka: wiele wariantów `set(...)`).
3. **Składanie `instructions`** — stringi w składni z dokumentacji.
4. **Iteracja** — `submit_to_hub` + korekty po feedbacku huba; ewentualnie `hardReset` przy chaosie konfiguracji.

## Wskazówki kursu

- Dwuetapowo: vision, potem tekst (oszczędność tokenów).
- Podejście reaktywne — feedback huba prowadzi korekty.
- Lekcja `02_05_sandbox` (code mode) to **osobny przykład architektury** — homework `drone` używa standardowej pętli ReAct + MCP z boilerplate.

## Zmienne środowiskowe

W `tasks/.env`:

- `OPENAI_API_KEY` lub `OPENROUTER_API_KEY`
- `HUB_API_KEY`
- `DRONE_MAP_URL` — URL mapy PNG
- opcjonalnie: `AGENT_MODEL`, `AGENT_MAX_ITERATIONS` (np. 30–50), `AGENT_ENABLE_PLANNING=false` (debug), `OM_PERSIST_DIR`

## Uruchomienie

```bash
cd tasks/s02e05
bun install
bun --env-file=../.env run run.ts
```
